# CC-Connect 官方安装配置指南（Windows 安全版）

## 项目是什么

CC-Connect 用于把本地 AI Agent 接入聊天平台。官方支持的 Agent 包括 Claude Code、Codex、Cursor、Gemini CLI、OpenCode、Kimi、Qoder、iFlow、Pi、Devin 和 GitHub Copilot CLI 等；聊天平台包括飞书、钉钉、Telegram、Slack、Discord、企业微信、微信个人号、QQ、Matrix 等。

官方仓库：`chenhg5/cc-connect`

## 一、安装前检查

在 PowerShell 中运行：

```powershell
node --version
npm --version
claude --version
codex --version
```

至少保证一个 Agent 可独立工作。

## 二、安装 CC-Connect

官方推荐 npm：

```powershell
npm install -g cc-connect
cc-connect --version
```

如果出现命令找不到：

```powershell
npm config get prefix
$env:Path
```

确认 npm 全局安装目录已经加入 Windows PATH。

## 三、用 Web UI 配置

```powershell
cc-connect web
```

浏览器管理界面可用于：

- 创建项目
- 选择 Claude Code、Codex 等 Agent
- 设置本地工作目录
- 添加聊天平台
- 管理服务商和模型
- 测试 Agent 对话

注意：`cc-connect web` 只做配置和打开网页，不会启动 CC-Connect 后台服务。

## 四、配置文件位置

默认：

```text
C:\Users\<用户名>\.cc-connect\config.toml
```

修改前备份：

```powershell
Copy-Item "$HOME\.cc-connect\config.toml" "$HOME\.cc-connect\config.toml.bak"
```

不要把真实配置提交到 GitHub。

## 五、最小配置示例

### Codex 项目

```toml
language = "zh"

[log]
level = "info"

[[projects]]
name = "my-codex-project"
admin_from = "YOUR_CHAT_USER_ID"

[projects.agent]
type = "codex"

[projects.agent.options]
work_dir = "C:/path/to/project"
mode = "suggest"
```

### Claude Code 项目

```toml
[[projects]]
name = "my-claude-project"
admin_from = "YOUR_CHAT_USER_ID"

[projects.agent]
type = "claudecode"

[projects.agent.options]
work_dir = "C:/path/to/project"
mode = "default"
```

不要默认使用：

```text
yolo
full-auto
bypassPermissions
```

## 六、常见平台

### 飞书

官方提供快捷配置：

```powershell
cc-connect feishu setup --project my-codex-project
```

飞书通常使用 WebSocket 长连接，不需要公网 IP。仍需确认机器人能力、消息权限、事件订阅和应用发布状态。

### Telegram

1. 在 Telegram 中联系 `@BotFather`。
2. 使用 `/newbot` 创建机器人。
3. 把 Bot Token 写入本机 `config.toml`。
4. 不要把 Token 发到公开群或提交 Git。

### Slack

通常需要：

- Socket Mode
- App-Level Token
- Bot Token
- `chat:write`、`im:history`、`im:read`、`im:write`、`app_mentions:read` 等权限

### 微信个人号

使用官方项目当前提供的 Weixin / ilink 接入方式。微信接口和登录方式可能变化，实际配置时以当前官方文档和程序 Web UI 为准。

## 七、启动

```powershell
cc-connect
```

不要关闭这个终端窗口，除非已经另外配置成 Windows 服务或计划任务。

## 八、首次测试

1. 从管理员聊天账号给机器人发送普通问题。
2. 确认 Agent 启动目录正确。
3. 让 Agent 执行只读任务，例如列出当前项目文件。
4. 确认未授权账号不能运行 `/shell`、`/dir` 等特权操作。
5. 再测试文件、图片或语音能力。

## 九、升级

```powershell
Copy-Item "$HOME\.cc-connect\config.toml" "$HOME\.cc-connect\config.toml.bak"
npm update -g cc-connect
cc-connect --version
```

升级后先做一次普通消息测试，再恢复长期运行。

## 十、排障

按以下顺序，不要一上来就重装：

```powershell
cc-connect --version
codex --version
claude --version
Test-Path "$HOME\.cc-connect\config.toml"
```

然后检查：

- TOML 语法
- 工作目录是否存在
- Token / App ID / Secret 是否有效
- 平台机器人权限及事件订阅
- 管理员用户 ID
- 网络代理和平台可访问性
- 启动日志中的第一条错误
