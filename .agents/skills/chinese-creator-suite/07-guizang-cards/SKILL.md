# 07｜鬼藏社交卡片与公众号封面

## 定位

把同一篇内容改造成适合平台阅读的视觉产品，而不是简单把长文切成几张大字报。每一页只承担一个观点、证据、步骤或情绪动作。

## 适用输出

- 小红书 / Rednote `3:4` 图文组图
- 社交媒体轮播卡片
- 微信公众号 `21:9` 头图 + `1:1` 分享卡
- 产品测评、教程、旅行、影视、AI 工具和数据复盘卡片
- 截图型说明图和文章配图

## 两套视觉系统

### Editorial 电子杂志

适合叙事、生活方式、旅行、阅读、影视和个人观察：

- 宋体或衬线大标题
- 纸张、墨水、留白、边注和拉引语
- 图片井、编号、账本式信息和克制配色

### Swiss 瑞士国际主义

适合产品、数据、工具、教程、AI 和方法论：

- 严格网格、发丝线和单一高饱和锚点色
- 巨大字号反差、KPI、矩阵、对比条和编号系统
- 冷静、工程化、信息密度高

同一套卡片默认只选一种系统，不随页乱切风格。

## 工作流

### 1. Intake

从现有上下文提取：

- 平台和比例
- 原始文章、字幕、文案或标题
- 目标读者和发布目的
- 用户已有照片、截图和必须使用的素材
- 风格偏好和硬性标题

如果只有文字、没有任何图片或截图，只问一次素材方案：用户图、公开授权图源或 AI 生图。用户选择后继续，不反复劝导。

### 2. 内容拆页

先建立页面计划：

```text
Page 01 / 封面 Hook / 主标题 / 主视觉
Page 02 / 问题或背景 / 证据
Page 03 / 核心判断 1 / 图或数据
Page 04 / 核心判断 2 / 对比或流程
Page 05 / 方法或清单
Page 06 / 结论与行动
```

小红书通常 5-9 页。每页一个核心信息；细节留给正文，不把整篇文章缩小塞进去。

### 3. 图像与截图

- 用户素材优先。
- 新闻、教程、测评和数据内容主动使用真实截图或证据图。
- 不拉伸截图，不让装饰压过可读性。
- 文字叠图时避开人物脸部和视觉主体。
- 使用网络图片时记录来源和授权信息。

### 4. 公众号封面对

- `21:9`：保留完整或接近完整标题、一个主视觉关系。
- `1:1`：从长标题提炼 4-10 字短标题，默认大字居中、少图或无图。
- 两张封面必须共享字体、色彩和视觉语言。

### 5. HTML 与导出

优先制作一个自包含 HTML，再通过浏览器或 Playwright 渲染 PNG：

```text
social-card-{slug}/
├── index.html
├── assets/
├── SOURCES.md
└── output/
    ├── 01-cover.png
    ├── 02-content.png
    └── ...
```

如果完整上游模板和校验脚本已安装，优先复制种子模板，不从空白 HTML 重写；运行溢出、字号和页脚碰撞检查后再交付。

## 质量检查

- 第一页是否在一眼内说明主题？
- 每页是否只有一个主要任务？
- 中文是否达到手机阅读字号？
- 是否出现模板重复、底部空洞或四横带拥挤？
- 真实截图和数据是否足够清晰？
- 卡片是否像平台原生内容，而不是 PPT 截图？
