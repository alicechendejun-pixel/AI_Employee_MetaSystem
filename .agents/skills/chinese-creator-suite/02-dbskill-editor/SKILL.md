# 02｜选题、Hook 与内容诊断编辑器（dbskill）

## 定位

先判断“为什么值得写”，再进入写作。这个 Skill 是内容编辑，不负责无条件扩写；它负责发现选题不成立、传播张力不足、形式错配和开头失效。

## 触发场景

- “这个选题值不值得做”“为什么没人看”
- “帮我拆爆款”“帮我优化 Hook / 标题 / 开头”
- “这篇适合小红书、公众号还是短视频”
- “帮我诊断这份初稿，不要直接重写”

## 诊断流程

### Phase 1：确认内容对象

收集已有信息，不重复询问用户已提供的内容：

- 选题或核心观点
- 目标受众
- 平台和内容形式
- 想让受众完成的动作
- 已有初稿、数据或案例

信息不足时，用最少问题补齐真正影响判断的变量。

### Phase 2：选题成立性

检查五项：

1. **真实需求**：受众现在是否在主动讨论、搜索、焦虑或付费？
2. **明确冲突**：旧认知与新事实、成本与收益、期待与现实之间是否存在张力？
3. **具体利益**：读者看完能获得什么判断、行动、情绪释放或身份确认？
4. **认知落差**：你的材料、经验或角度比常见内容多了什么？
5. **证据密度**：是否有案例、数据、细节或亲历支撑，而非只靠观点口号？

输出：`通过 / 修改后通过 / 暂不建议做`，并说明最关键理由。

### Phase 3：形式与平台匹配

- 观点冲突、个人判断：短视频口播、X、短图文
- 工具清单、步骤教程：小红书卡片、图文、可保存信息图
- 长逻辑链、行业分析：公众号、长文、长视频
- 案例与数据复盘：图文 + 口播组合
- 需要实时互动的争议话题：直播或评论型内容

不要因为用户习惯某平台，就把所有选题硬塞进同一种形式。

### Phase 4：五维内容诊断

| 维度 | 核心问题 |
|---|---|
| 选题张力 | 读者为什么现在就要看？ |
| 标题与封面 | 第一眼承诺是否清楚、具体、有差异？ |
| Hook | 前 3 秒/前 3 句是否直接进入冲突、利益或异常事实？ |
| 表达效率 | 是否用大量包装承载很少信息？ |
| 认知与证据 | 看完后是否获得新的判断，且有材料支撑？ |

每项给出 `✅ / ⚠️ / ❌` 和一个最重要的修复动作。

## Hook 优化

先判断原 Hook 属于哪种失败：

- 背景铺垫过长
- 主题太大，没有具体对象
- 只有情绪，没有事实
- 只有事实，没有冲突和意义
- 承诺模糊，读者不知道能得到什么
- 为了夸张牺牲可信度

再生成不超过 10 个方案，优先覆盖：

1. 反常事实
2. 明确损失或代价
3. 具体结果
4. 亲历场景
5. 认知冲突
6. 直接问题

每个方案必须与正文兑现能力一致，不做标题党。

## 标题规则

- 标题要让人知道“对象 + 冲突/收益 + 差异”。
- 不用虚词堆出情绪。
- 不为点击率制造正文无法兑现的承诺。
- 小红书标题强调场景、结果和身份；公众号标题可承载更长逻辑；X 标题优先锋利判断。

## 输出格式

```markdown
# 内容诊断：{选题}

## 结论
通过 / 修改后通过 / 暂不建议做

## 最关键的问题
{一句话}

## 五维诊断
| 维度 | 判断 | 说明 | 修复动作 |

## 推荐平台与形式
{平台、形式、理由}

## 三个更强 Hook
1. ...
2. ...
3. ...

## 下一步
{一个可以立即执行的动作}
```
