# GEO 输出格式规范

默认使用 Markdown，除非用户指定其他格式。

## 1. 改造后的完整文章

```markdown
# 优化后的文章标题

## 核心要点摘要
- ...
- ...
- ...

## 正文

### ...

## 结论

## FAQ
**Q1：...**
A：...

---
**内容更新时间**：YYYY-MM-DD 或 [建议补充更新时间]
**证据边界**：说明哪些来自原文、哪些经过外部核验、哪些仍需补充。
```

只有文章足够长、目录确实有助于浏览时才添加目录。

## 3. 改造执行说明

按用途分组：

- `结构层面`：标题、摘要、FAQ、表格、步骤、段落重排
- `证据层面`：来源标签、引语建议、数据口径、可追溯事实
- `表达层面`：过渡、段落平衡、术语定义、可读性
- `语义层面`：实体展开、问题覆盖、邻近领域连接

只有能够可靠统计时才使用具体数量，否则说明具体位置和示例。

## 4. 需要用户补充或确认的内容

```markdown
1. 第 X 节：[建议补充来源] ...
2. 第 X 节：[建议补充数据口径] ...
3. 第 X 节：[建议确认术语] ...
```

没有补充项时，明确说明所有内容均来自原文或已核验来源。

## 5. 改造风险提示

只说明真实存在的风险，例如：

- 原文数据没有来源，已保守保留或标注。
- 某些术语解释基于行业通识，需要确认具体行业口径。
- 深度重写改变了段落顺序，但保留了核心观点。
- 外部来源具有时效性，需要后续复核。
