# GEO文章AI友好化改造

## 适用场景

当用户提供已有文章正文或本地文件路径，并要求进行 GEO、生成式搜索优化、AI 搜索友好化改造、引用准备度提升、结构化重写或改造前后评分时使用。

支持的输入包括：

- 直接粘贴的文章正文
- `.md`
- `.txt`
- `.html` / `.htm`
- `.docx`
- `.pdf`（当前环境能够提取正文时）

如果无法读取文章，明确说明阻塞原因，并要求用户粘贴正文。

## 不适用场景

不要用于：

- 只有选题、没有原文的从零写作
- 法律、医疗、金融或广告合规判断
- 承诺 GEO 排名、引用率、收录或推荐结果
- 攻击竞争对手、删除内容或操纵平台
- 编造统计数据、来源、专家引语、案例、报告或机构
- 只需要泛泛 GEO 策略，而不需要改造具体文章

## 必读资料

开始执行前，根据任务需要读取：

- `参考资料/GEO文章改造方法.md`：加权方法、证据规则、改造优先级与验证标准。
- `参考资料/GEO输出格式规范.md`：最终 Markdown 结构、评分表和改造说明格式。
- `报告/输出风险说明.md`：常见风险与交付前检查。

## 工作流

1. 读取原文，保留原始标题、核心观点、事实、案例、语气和结论。默认使用 `standard` 标准深度。
2. 盘点核心观点、证据、数据、引语、案例、结构、实体、术语和可复用段落。
3. 按 GEO 加权维度评估原文，并对来源、数据口径和引语缺失保持保守评分。
4. 按优先级改造：证据、结构、表达流畅度、语义密度、权威信号、术语一致性、鲁棒性、跨领域连接和可读性。
5. 对缺乏依据但值得补充的内容，使用 `[建议补充...]`，不得直接写成事实。
6. 输出改造后的完整文章、评分报告、改造说明、补充清单和风险提示。
7. 交付前核对：原意未改变、没有编造、高权重项确实改善、文章仍然自然可读。

## 证据纪律

需要说明来源边界时，使用以下标签：

- `原文支持`
- `外部已核验`
- `建议补充`

如果涉及当前法律、平台规则、市场数据、排名、产品事实或最新研究，必须使用当前的一手或权威来源核实后才能加入。无法核实时，保留为 `建议补充`。

## 改造深度

- `light`：只优化证据准备度和结构，尽量贴近原文风格。
- `standard`：默认模式，保守地改善全部加权维度。
- `deep`：用户明确要求研究增强时，允许更大幅度重构，并补充经过核验的外部证据。

长文超过约 5000 个中文字或 3000 个英文单词时，分章节处理，最后统一输出总评分和改造报告。

## 默认输出

默认输出一份结构化 Markdown，包含：

1. `改造后的完整文章`
2. `GEO优化度评分`
3. `改造执行说明`
4. `需要用户补充或确认的内容`
5. `改造风险提示`

用户要求生成文件时，保存 Markdown 文件并返回文件路径。

## 交付前检查

- 所有原文核心观点是否保留？
- 是否新增了原文没有且未经核实的数据、引语、案例或报告？
- 每个关键事实是否可追溯或已标注来源边界？
- 证据、数据、可引用性、结构和表达这五个高权重项是否真实改善？
- FAQ 是否只回答原文能够支持的问题？
- 是否避免关键词堆砌和过度优化？
- 评分是否有依据，而不是为了显得效果好而虚高？
- 改造后的文章是否仍然像正常的人类文章？
