# 06｜Punk 中文封面生成器

## 定位

封面不是装饰，而是内容的第一层信息。目标是在第一眼说明主题，在第二眼呈现冲突或视觉隐喻，同时保证中文标题完整、清晰、可读。

## 触发场景

- “给这篇文章做封面”“生成公众号首图”
- “做小红书 3:4 封面”“做 X 长文封面”
- “根据长文提炼封面标题和视觉方案”
- “只输出可复用的生图提示词”

## 平台比例

| 平台 | 默认比例 |
|---|---|
| 小红书 / Rednote | `3:4` |
| 微信公众号头图 | `2.35:1` |
| X / Twitter | `5:2` |
| 自定义 | 严格使用用户给出的比例 |

用户已给平台或比例时，不重复询问。用户说“你决定”时自动选择。

## 工作流

### 1. 从正文提炼封面字段

只提炼，不把整篇文章塞进提示词：

- 完整主题
- 高冲击短标题
- 可选副标题
- 1-3 句内容摘要
- 视觉主体
- 目标读者
- 情绪
- 视觉隐喻
- 禁止出现的元素

### 2. 选择单一风格

每次只使用一种明确的视觉系统，不混搭。无指定风格时，基于内容推荐 3 个方向并各写一句理由；用户说“自动”时直接选最适合的一种。

可优先考虑：

- 编辑杂志 / 报纸排版
- 黑白极简 / 前卫几何
- 撕纸拼贴 / 旧报纸纹理
- 巨型中文标题 / 高对比色块
- 商业报告 / 策略咨询视觉
- 科研手账 / 技术图谱

### 3. 编译完整提示词

提示词必须是一个完整封面 brief，而不是“任务说明 + 风格词”简单拼接。至少包含：

```text
用途与比例
完整标题与层级
内容摘要
视觉主体与构图
图文关系
中文排版要求
材料、色彩、纹理与空间逻辑
必须保留的风格锚点
禁止项
最终质量标准
```

### 4. 中文标题规则

- 主标题必须准确，不能改错事实和专有名词。
- 标题太长时，用“大短标题 + 小完整标题/副标题”保留完整含义。
- 禁止标题被主体严重遮挡、裁切或写错。
- 不放无意义英文、乱码、课程广告和电商促销元素。

### 5. 输出文件

在项目中保存：

```text
punk-assets/punk-cover/{slug}/prompts/cover.md
punk-assets/punk-cover/{slug}/cover.png
```

只有图像工具明确返回当前图片文件时才保存 `cover.png`，不扫描其他目录猜测产物。

### 6. 生成图片

用户要求生成图片且当前环境有图像生成工具时，直接调用图像工具。用户只要提示词时，不生成图片。

## 质量禁区

- PPT 模板感、网课封面感、电商广告感
- 无关装饰、过度堆字、标题缺失或错字
- 多方案拼成九宫格或联系表
- 同时混合多种视觉风格
- 把正文原文缩小塞满画面
