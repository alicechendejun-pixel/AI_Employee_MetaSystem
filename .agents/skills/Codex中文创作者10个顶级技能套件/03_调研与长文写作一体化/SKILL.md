# 03｜调研与长文写作一体化（Content Research Writer）

## 定位

写作不是从空白页开始，而是从材料密度开始。本 Skill 把研究、提纲、写作、引用和审稿连接成一个流程，同时保留作者自己的观点与语言。

## 适用场景

- 行业、公司、产品、AI、金融和趋势分析
- 公众号文章、博客、Newsletter、知识教程
- 需要数据、出处、案例和反方观点的深度内容
- 用户已有素材，需要整理成有逻辑的长文

## 工作流

### 1. 建立写作任务书

从现有上下文提取：

- 核心问题与主张
- 目标读者
- 文章目的：解释、说服、复盘、教育或销售
- 平台、长度和语气
- 已有材料和必须引用的来源
- 不能写错或不能公开的内容

### 2. 研究拆解

把主题拆为：

1. 必须核实的事实
2. 必须找到的数据
3. 需要案例支撑的观点
4. 可能存在争议的判断
5. 读者最可能提出的反问
6. 需要当前联网确认的信息

外部事实必须使用可靠来源。对当前新闻、价格、法律、模型、政策、公司和产品信息重新检索，不使用示例数字冒充事实。

### 3. 建立研究笔记

每条材料记录：

```text
结论：
证据：
来源：
发布日期：
事件发生日期：
可支持文章哪一段：
限制或反方解释：
```

区分：事实、来源观点、作者判断和推测。

### 4. 共同搭建提纲

提纲至少包含：

- Hook：异常事实、冲突、具体场景或核心问题
- 背景：只保留读者理解论点所需信息
- 主体：每节一个判断，每个判断配证据或案例
- 反方：处理最强反驳，而不是虚构稻草人
- 结论：给出新的判断或行动，不重复全文
- 研究缺口：仍需补证据的地方

### 5. 分段写作与审查

每一节检查：

- 核心句是否清楚
- 证据是否真正支持结论
- 是否混淆相关性与因果
- 是否有重复、跳跃或材料堆砌
- 是否保持作者原有语气
- 是否需要更具体的例子

### 6. 引用管理

根据平台选择脚注、链接、行内引用或文末参考资料。引用必须能追溯到真实来源，不编造作者、报告、链接或统计数字。

### 7. 终稿检查

- 标题和正文是否兑现同一承诺
- 所有可核实主张是否有来源
- 日期、数字、公司名和专有名词是否准确
- 是否存在没有证据的宏大总结
- 是否保留作者判断，而不是变成资料汇编
- 是否经过 `creator-stop-slop` 做最终文风清洗

## 默认输出

```markdown
# 写作任务书
# 研究问题清单
# 关键材料与来源
# 文章提纲
# 正文
# 事实核查清单
# 参考资料
```

用户只要求其中一个阶段时，不强制输出全套。
