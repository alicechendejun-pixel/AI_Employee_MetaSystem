# 怎么调用这套 AI 员工系统不烧额度

## 核心规则

1. 每次只选 1 个主责角色。
2. 复杂任务最多组 3 个角色：主责、审核、执行。
3. 不要把整个 `01_SKILL角色库` 拖进模型。
4. 先打开 `99_索引与调用规则/角色总索引.md` 找角色。
5. 再打开目标角色的 `角色说明.md`。
6. 只有真正要让它工作时，才打开同文件夹的 `ROLE.md`。
7. 给角色的资料要压缩成任务背景、目标、已有材料、输出格式、限制条件。
8. 让角色先给任务清单，再决定是否继续细化，避免一次性生成过多内容。

## 推荐调用格式

```text
请读取这个角色文件：<某个角色文件夹>/ROLE.md。
你现在只扮演这个角色，不要调用其它角色。

任务背景：
目标：
已有材料：
输出格式：
限制条件：
先给我一个可执行清单，再开始处理。
```

## 什么时候组队

- 做 YouTube：内容策划 + 视频优化 + 封面/视觉，必要时再加数据分析。
- 做跨境电商：跨境电商运营 + PPC 投放 + 供应链物流 + 法务合规。
- 做外贸销售：销售开发 + 提案策略 + LinkedIn 内容 + 合同审查。
- 做产品上线：产品经理 + 快速原型工程师 + UI 设计 + 测试。

## 省额度提醒

把角色当员工，不要当群聊。员工越多，会议越贵。
