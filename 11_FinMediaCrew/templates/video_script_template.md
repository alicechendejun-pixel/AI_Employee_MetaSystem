# 视频口播脚本模板

**用于**：YouTube / B 站 / 抖音 / 微信视频号。
**目标**：8-12 分钟，约 2500-3500 字。

---

## 1. 标题（3 型，YouTube 算法）

| 类型 | 公式 | 示例 |
|---|---|---|
| 好奇型（**首选**） | 反直觉问题 | "我让 4 个 AI Agent 给同一只股做研报 — 3 个翻车" |
| 直接搜索型 | 关键词 + 解释 | "FinRobot 工作流详解 + 私募风控视角的 6 个补强点" |
| 利益型 | 时间 + 结果 | "2026 中产 AI 投资必学：用 4 个 Agent 6 分钟完成一份研报" |

## 2. 封面

- 视觉：分析师本人画面 + 数据图叠加
- 文字：**3-5 字荧光绿**
- 手机端测试：拇指大小可读

---

## 3. 完整脚本结构（按时间）

### 【0:00–0:30 · 钩子】

**钩子公式**：1 个真实数字 + 1 个反差 + 1 个悬念。

示例：
```
我去年用 FinRobot 给 NVDA 做了一份研报 —
它给的目标价是 $310。

我同时用 TradingAgents 跑了同一只股 —
它给的目标价是 $250。

差 $60。

我想知道为什么。

3 天后我搞清楚了 —
这 $60 的差距，
藏着 AI 投资里最值钱的一件事。

今天这一集，我把它拆给你看。
```

### 【0:30–1:30 · 铺垫 + 身份锚定】

⚠️ **每集必出**身份锚定，让算法识别"垂直创作者"：

```
我是分析师 Merci / Alice Chen：
前财经记者 + 前私募合伙人风控 + 前情感咨询师 + AI 重度玩家。

我做这个频道，专门研究 ——
在 AI 爆发、经济下行同时发生的时代，
一个普通人怎么用 AI 重新组织自己已经有的金融判断力。

这一集是《建设性金融实验》第 X 集。
```

### 【1:30–3:30 · 现象诊断】

把"3 路分析"讲清楚 ——
- 哪个模型怎么算的。
- 关键假设是什么。
- 为什么差距是 $60 而不是 $6。

视觉建议：屏幕分 3 栏，每栏一个模型的输出。

### 【3:30–6:30 · 深挖分歧 + 框架】

**这一段是视频的核心**：
- 拆开关键分歧变量。
- 用 1 张图 / 1 张表呈现。
- 把"私募风控视角"作为差异化锚位 ——
  "如果是我当年管基金，我会怎么处理这个分歧"。

### 【6:30–9:00 · 4 个建仓信号 / 4 个动作】

类似 EP01-EP10 的结构：

```
动作一：{具体动作}
动作二：{具体动作}
动作三：{具体动作}
动作四：{具体动作}
```

每个动作要有：
- 1 个真实数据 / 案例
- 1 个具体可执行步骤
- 1 个心理 / 操作上的难点

### 【9:00–10:30 · 收束 + 系列预告】

```
我们 4 个 AI 跑下来 —
结论不是"NVDA 一定能涨到 $310"。

是 ——
3 个独立模型有共识的地方，可以信。
3 个独立模型打架的地方，是真变量，不要漏看。
3 个独立模型都没考虑的地方，是你的独家机会。

接下来 EP X 我会讲 ——
怎么把这个框架套到 TSLA / META / BTC 上。
```

### 【10:30–11:00 · 最后一句 + CTA】

```
你不是没分析能力 —
你只是没让 4 个 AI 替你做交叉验证。

订阅，一起实验。
```

## 7. 录制 SOP

1. **前 30 秒必须重剪 3 遍** — 是钩子能否抓住的关键。
2. **每 90 秒切换 1 次画面** — 防止留存衰减。
3. **关键术语停顿 1.5 秒** — 让观众"安装补丁"。
4. **避免感叹号堆叠** — 用克制语气讲深度内容。
5. **录完不剪超过 1 小时** — 防止完美主义。

---

## 8. 系列互锁规则

每集必须：
- 引用 2-3 集历史（"EP X 讲过 Y 概念"）
- 预告下集（"EP X+1 我们讲 Z"）
- 挂母系列（"这是《建设性金融实验》第 X 集"）

让任何一集都能当"第一集"看 — 但每集都引向更多集。
