## 📅 定时跑（让它自动每天发简报）

详见 [`deployment/cron.md`](./deployment/cron.md)。

**最简单做法**（Windows）：

```powershell
$action = New-ScheduledTaskAction -Execute "python" `
  -Argument "crew/main.py --ticker NVDA --mode brief_only" `
  -WorkingDirectory "G:\我的云端硬盘\01-obsidian\04 AI知识库\09 金融记录共享\FinMediaCrew"

$trigger = New-ScheduledTaskTrigger -Daily -At 7:00AM

Register-ScheduledTask -TaskName "FinMediaCrew_Daily" `
  -Action $action -Trigger $trigger -RunLevel Highest
```

每天早 7:00 自动跑 + 推飞书。

## ❓ 常见问题

### Q1: 报错 "FinRobot 不在预期路径"
**A**: 检查 `.env` 里 `FINROBOT_ROOT` 路径是否对。Windows 路径用反斜杠：`G:\...`

### Q2: TradingAgents 跑了 30 分钟没出来
**A**: 这是正常的（多 agent 辩论确实慢）。如果想加快：
```bash
TRADINGAGENTS_DEBATE_ROUNDS=1 python crew/main.py --ticker NVDA --mode full
```

### Q3: 输出的内容质量不够
**A**: 3 个可能：
1. **LLM 模型太小** — 在 `.env` 里把 `FINMEDIA_LLM_MODEL=gpt-4o-mini` 换成 `gpt-5` 或 `claude-4-7-sonnet`
2. **数据不够** — 检查 `raw_data.json` 是否完整
3. **NarrativeAgent 钩子不强** — 手动改 `hook.md` 然后用 `--mode content_only` 重跑

### Q4: 不想全跑，只想生产某一种内容
**A**: 用 `--mode content_only --report-path` 指定已有研报：
```bash
python crew/main.py --ticker NVDA \
    --mode content_only \
    --report-path "examples/output_NVDA_2026-05-28/cross_analysis.json"
```

### Q5: API 费用太贵
**A**:
- 把 LLM 换成 `gpt-4o-mini`（便宜 10x）
- 把 `TRADINGAGENTS_DEBATE_ROUNDS` 设 1
- 用 `--mode brief_only` 跑（不开 cross_analysis）

### Q6: 上游 SKILL 没装好
**A**: 没装好不影响 — 工具会自动 skip + 用其他 SKILL 兜底。但**至少**装好 FinRobot 或 TradingAgents 中的 1 个。

### Q7: 想加新平台分发（公众号 / 小红书）
**A**: 看 `crew/tools/`，仿照 `feishu` 的写法新增。
